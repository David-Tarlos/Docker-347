Erklärung
Erstellen und starten Sie einen Container mit dem Befehl docker run -d. Kommentieren Sie was mit dem Container geschieht in 3-5 Sätzen.  wurde das Image automatisch heruntergeladen? Konnte es starten?:
Antwort: Das Ubuntu-Image wurde automatisch von Docker Hub heruntergeladen, da es lokal noch nicht vorhanden war. Der Container wurde erstellt und gestartet, hat sich aber sofort wieder beendet (Status "Exited"). Grund dafür ist, dass das Ubuntu-Image keinen langlaufenden Hauptprozess hat — anders als z.B. nginx, das einen Webserver-Prozess mitbringt. Ein Container lebt nur so lange wie sein Hauptprozess; ohne aktiven Prozess endet er sofort. Das Flag -d (detached) macht hier keinen Unterschied, weil es gar nichts gibt, das im Hintergrund laufen könnte.

Antwort für Teil 2 (docker run -it ubuntu)
Erstellen und starten Sie einen Container mit dem Befehl docker run -it. Kommentieren Sie was nun geschieht in 3-5 Sätzen.
Mit den Flags -i (interactive) und -t (TTY) wird beim Start des Containers eine interaktive Shell geöffnet. Man landet direkt im Bash-Prompt des Ubuntu-Containers (z.B. root@abc123:/#) und kann Linux-Befehle ausführen, als wäre man auf einem normalen Ubuntu-System. Die Bash ist jetzt der Hauptprozess des Containers und hält ihn am Leben, solange die Shell offen bleibt. Mit dem Befehl exit wird die Shell beendet — und damit auch der Container, weil der Hauptprozess endet. Das zeigt, dass Ubuntu nur dann sinnvoll als Container nutzbar ist, wenn man ihm aktiv einen Prozess (wie eine Shell) zur Verfügung stellt.
